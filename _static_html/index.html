What is LXD?

LXD (pronounced lex-dee) is a modern, secure, and powerful system container and virtual machine manager developed by Canonical. It provides a unified experience for running and managing full Linux systems inside containers or virtual machines. LXD scales from a single machine to a full data center rack, making it suitable for both development and production workloads.

With LXD, you can easily set up a system that feels like a small private cloud, running any type of workload efficiently while optimizing your resources.

Key Features

System Containers

Run full Linux systems as containers with minimal overhead. LXD containers provide the same user experience as virtual machines but with lower resource requirements and faster startup times.

Virtual Machines

LXD also supports full virtual machines when hardware-level isolation is needed, providing a unified management interface for both VMs and containers.

Image-based Deployment

LXD uses pre-built images for a wide variety of Linux distributions, making it easy to deploy any supported distribution in seconds.

Advanced Networking

Support for multiple network interfaces, bridges, macvlan, SR-IOV, OVN and more, allowing complex network topologies for your containers and VMs.

Storage Management

Flexible storage options including ZFS, Btrfs, LVM, Ceph, and directory-based storage with features like snapshots and quotas.

Common Use Cases

Development Environments

Create isolated, reproducible development environments that mimic production. Each developer can have their own instances without interfering with others.

DevOps CI/CD Testing

Hosting & Application Deployment

Run multiple applications in isolated environments on the same hardware, increasing density and reducing costs compared to traditional VMs.

Websites Microservices Cloud Native

Edge Computing

Deploy multiple workloads on edge devices with minimal overhead, perfect for IoT gateways and remote computing nodes.

Quick Start Guide

Installing LXD on Ubuntu 
$ sudo snap install lxd
$ sudo lxd init
Creating your first container 
$ lxc launch ubuntu:22.04 my-container
$ lxc exec my-container -- bash

More examples

Creating a VM instead of a container 
$ lxc launch ubuntu:22.04 my-vm --vm
$ lxc console my-vm
Creating a custom profile 
$ lxc profile create development
$ lxc profile edit development

Advanced Features

Clustering

LXD clustering allows you to manage containers and virtual machines across multiple hosts as if they were local. This provides high-availability and load balancing capabilities for your infrastructure.

Key benefits include:

  • Distributed storage and compute resources
  • Transparent instance migration between nodes
  • Single management endpoint for your entire cluster
Learn more about clustering

Storage Backends

LXD supports various storage backends to meet different performance and feature requirements:

  • ZFS: Advanced features like compression, quotas, and snapshots
  • Btrfs: Similar to ZFS with built-in copy-on-write
  • LVM: Logical Volume Management for flexibility
  • Ceph: Distributed storage for clustering
  • Directory: Simple storage option requiring no configuration
Explore storage options

Security Features

LXD provides comprehensive security controls including:

  • AppArmor and Seccomp profiles for container isolation
  • Resource limits (CPU, memory, disk I/O, network)
  • Role-based access control
  • TLS encryption for all communications
  • Project-based multitenancy
